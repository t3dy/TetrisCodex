Shared Alchemy Block Engine

The alchemical block system should outlive any one game mode. Balance Tetris, Space Taxi, Rune Bloom, pinball labs, flow puzzles, construction kits, and future experiments can all use the same alchemical matter while keeping their own controls and win conditions.

Layer Model

layer 0

Sources

Timmermann, Ashmole, Maier, Ripley, Norton, Emerald Tablet, QueryPat, and local database records.

layer 1

Block Engine

Defines alchemical properties: mass, friction, volatility, fixity, combustibility, process, principle, source tags.

layer 2

Mode Adapters

Convert shared blocks into Balance cells, Taxi cargo, Rune tiles, pinball bodies, lab reagents, or codex cards.

layer 3

Mode Runtime

Owns input scheme, board geometry, camera, scoring, objectives, and failure rules.

Shared Block Example

const block = AlchemyBlockEngine.makeAlchemyBlock({
  processId: 'cancer',
  principleId: 'mercury'
});

const balanceCell = AlchemyBlockEngine.adapters.toBalanceCell(block);
const taxiCargo = AlchemyBlockEngine.adapters.toTaxiCargo(block);
const runeTile = AlchemyBlockEngine.adapters.toRuneTile(block);
