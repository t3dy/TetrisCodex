Design Rationale

Each mode began as a prompt or correction. This page records the originating request, how it was interpreted, what implementation choices were made, and what alternate designs remain open.

core game

Balance Tetris

Original prompt

“There is a board game version of tetris where the user stacks up tetris like blocks on a balancing platform. I'd like to create a digital version of this game.”

Interpretation

A deterministic 2D balance-platform game with tetromino-like blocks, torque, tilt, and multiple platform geometries.

Choices made

  • Grid-snapped blocks.
  • Torque math instead of full rigid bodies.
  • Mouse and keyboard controls.
  • Early-placement grace after playtest feedback.

Other choices

Full 3D physics, real-time-only Tetris, or pure puzzle levels with no random queue.

alchemy

Alchemy Balance

Original prompt

“I still want the balance tetris where you can profitably tip alchemical substances off the appropriate side of the platform.”

Interpretation

Tipping should become a scoring verb, not only a failure condition.

Choices made

  • Planetary exit-side affinities.
  • Salt/Sulphur/Mercury material behavior.
  • Group slide scoring.
  • Wrong-side penalties.

Other choices

Cosmetic alchemy, recipe-only alchemy, or a separate non-physics reaction board.

world grammar

Combinatorial Palette

Original prompt

“The combinatorial system where the player clicks on a zodiac sign... or adding a sulfur salt mercury button push...”

Interpretation

Zodiac signs select alchemical process/apparatus; Tria Prima buttons select material quality.

Choices made

  • Zodiac-alone apparatus.
  • Salt as fixed matter.
  • Sulphur as oily/combustive matter.
  • Mercury as volatile matter.

Other choices

Planet-only grammar, card inventory, multi-step recipes, or separate apparatus/material palettes.

separate experiment

Rune Bloom

Original prompt

“I like the idea of rune bloom but you can fire off dozens of game modes like that...”

Interpretation

A separate tile-placement experiment, not a replacement for Balance Tetris.

Choices made

  • Standalone page.
  • Glyph placement.
  • Bloom clearing.
  • Blight pressure.

Other choices

Fold it into Balance Tetris, make it decorative, or prioritize many variants before returning to balance play.

legend mode

Wild Hunt

Original prompt

“Create a system of emoji graphics for the wild hunt blocks... and create conceits for game levels in that mode based on the scholarship.”

Interpretation

A folklore chronicle mode where date tokens make unstable apparitions, hounds, hosts, and riders legible.

Choices made

  • Hound, hoofbeat, apparition, dead host, date, mirror hunt, huntsman, and alembic tiles.
  • Chronicle scoring.
  • Omen beats.
  • Credibility meter.

Other choices

A chase game, visual novel, linear manuscript adaptation, or night-flight VALIS Taxi route.

PKD modes

QueryPat / PKD

Original prompt

“Search through my QueryPat database for ideas for balance tetris game modes inspired by Philip K Dick stories.”

Interpretation

Use PKD motifs as mechanics: entropy, hidden reality, split perception, precognition, kipple, artificial life, and revelation.

Choices made

  • Ubik anti-entropy.
  • Scanner scramble.
  • High Castle hexagrams.
  • Minority Report queue play.
  • VALIS beam.

Other choices

A direct adaptation, text-heavy story mode, hidden modifiers only, or no PKD branch.

vehicle mode

VALIS Space Taxi

Original prompt

“I like the idea of the valis beam and I'm reminded of a space taxi old school game...”

Interpretation

A thrust-and-landing game where a ship docks with alchemical cargo on tilting pads and uses a VALIS beam.

Choices made

  • Standalone prototype.
  • Gravity, wind, hull, fuel, docking, beam heat.
  • Emoji cargo/passengers.
  • Cascade risk.

Other choices

Pure lunar lander, beam weapon game, or Balance Tetris skin with no flight model.

rotation puzzle

Alchemical Three

Original prompt

“I want various game modes to plug into the tetris rotate mechanic... a 3 vikings sort of game where players control characters that can pick up and sometimes rotate... the blocks.”

Interpretation

Rotation becomes a shared verb owned by characters, tools, and matter rather than only falling pieces.

Choices made

  • Three alchemical specialists.
  • Carry/drop grid puzzle.
  • Actor-gated block rotation.
  • Shared toCarryBlock adapter.

Other choices

Literal Viking roles, continuous platforming physics, two-character lifting, or a pure editor-only rotation tool.

puzzle branch

Moat Drawbridge

Original prompt

“There could be puzzles like get across the moat where the player has to build a stack tall enough and flat enough...”

Interpretation

A Balance Tetris puzzle family where the stack must become a functional bridge surface.

Choices made

  • Flatness objective.
  • Span/crossing concept.
  • Moat stone assets.
  • Two-fulcrum bridge logic.

Other choices

A separate bridge-builder, freeform beams, scripted bridge animation, or decorative crossing.

authoring

Level Builder

Original prompt

“I want a construction kit where the user can create levels for each of our game modes...”

Interpretation

The arcade needs a shared WYSIWYG editor and stable JSON schema.

Choices made

  • Block painting.
  • Platforms, pads, receivers, objectives.
  • Physics sliders.
  • Import/export JSON.

Other choices

Code-only schema, separate editors per mode, drag-first editor, or backend storage.

creator pipeline

Asset Studio / Website

Original prompt

“We'll also need to build other dev tools... and a website explaining all the game modes and assets.”

Interpretation

Assets need editable definitions and source notes, while users need a website explaining the game world.

Choices made

  • Asset Studio.
  • Game Guide.
  • Dev tools roadmap.
  • Asset JSON schema.

Other choices

Markdown-only docs, a full CMS, generated pages first, or postponing tools.

playability

Fairness / Instructions

Original prompt

“The game modes should all have instructions explaining what the player is expected to do so they can understand what a good move might be.”

Interpretation

Difficulty was not only tuning; modes needed onboarding and actionable first-move guidance.

Choices made

  • Early-placement grace.
  • Mode instruction cards.
  • Help overlay guidance.
  • Wild Hunt torque softening.

Other choices

Only lower difficulty, remove early failure entirely, or put all instruction in a separate manual.
